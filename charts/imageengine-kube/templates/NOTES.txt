({{ include "imageengine.fullname" . }}-osc-0 .. -osc-{{ sub (int .Values.objectStorageCache.replicaCount) 1 }}),

Heads up: service.type is "{{ .Values.service.type }}" and ingress.enabled is
false. The chart is installed but the edge isn't reachable from outside the
cluster. To expose it, either:

  - Set service.type=LoadBalancer (cloud LB controller required), or
  - Set ingress.enabled=true and provide ingress.hosts (ingress controller
    required), or
  - Use kubectl port-forward for testing:

    kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "imageengine.fullname" . }}-edge 8080:{{ .Values.service.port }}

{{- end }}

This BACKEND-ONLY install has no edge or Varnish. Point each remote frontend
tier's varnish.ieBackends at this stack's backend Service address:

  kubectl --namespace {{ .Release.Namespace }} get svc {{ include "imageengine.fullname" . }}-backend

{{- end }}

  WARNING: the ImageEngine backend is being exposed OUTSIDE the cluster. Its
  listener (:8000) serves UNAUTHENTICATED admin endpoints (POST
  /api/v1/loglevel, /api/v1/originreturn) and Varnish reaches it over PLAIN
  HTTP. It must NOT be reachable from the public internet.

    - ingress.enabled routes external traffic to the backend on this
      backend-only install, and provider presets may make that ingress

      backend.service LoadBalancer; if you must use an ingress, force it
      internal/private and restrict it.
{{- end }}
